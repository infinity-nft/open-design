# /start — старт рабочей сессии

Проверяет всё что нужно перед тем как сесть кодить: ветка, чистота, свежесть upstream, рабочее окружение.

## Алгоритм

Делать параллельно где возможно (один блок Bash-вызовов):

1. **Снапшот текущего состояния:**
   ```bash
   git status --short
   git branch --show-current
   git log --oneline -3
   ```

2. **Проверить свежесть upstream** (без изменения локального состояния):
   ```bash
   git fetch upstream
   echo "--- main vs upstream/main ---"
   git log --oneline main..upstream/main | head -5
   echo "--- local-main vs main ---"
   git log --oneline main..local-main | head -10
   ```

3. **Проверить рабочее окружение:**
   ```bash
   node --version
   pnpm --version
   ```
   Если node не ~24 — упомянуть в отчёте (warning, не блокер).

## Анализ и отчёт

Сделать короткий summary в формате:

```
📍 Текущая ветка: <name>
🔀 Состояние:
   - Незакоммиченных правок: <N файлов / clean>
   - main vs upstream/main: <X коммитов отстаёт / up-to-date>
   - local-main vs main: <Y коммитов своих / sync>

✅ Что готово:
   - <список>

⚠️ Что требует внимания:
   - <если есть отставание от upstream — предложить /sync>
   - <если есть незакоммиченные правки — предложить commit/stash>
   - <если не на local-main и собирается работать — предложить переключиться>

🎯 Что делать дальше:
   - Конкретное предложение исходя из состояния
```

## Решения по умолчанию

- **Если на `main`** и нет работы — предложить `git checkout local-main`.
- **Если main отстаёт от upstream** более чем на 5 коммитов — настойчиво предложить `/sync` перед началом работы.
- **Если есть незакоммиченные правки** в важных файлах (`apps/`, `packages/`) — НЕ синкать без явного согласия.
- **Если рабочее дерево чистое и всё свежее** — кратко сообщить что готовы к работе и спросить что делаем.

## Не делать

- Не запускать pnpm install автоматически — это медленно и обычно не нужно.
- Не делать pull/merge без подтверждения.
- Не запускать длинные таски (test, build) — пусть пользователь решит когда.

## Контекст проекта

Структура форка (см. memory `project_fork_structure.md`):
- `main` — зеркало `upstream/main` (nexu-io/open-design)
- `local-main` — рабочая ветка с кастомизациями
- Для синка: `/sync`
- Для PR в upstream: `/contribute`
